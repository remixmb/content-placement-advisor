Main SOM Taxonomy Audit and Placement Documentation

Executive summary

This work documents how taxonomy and section context drive content placement across the Yale SOM website. The documentation is the primary deliverable. The local Content Placement Advisor is a supporting prototype used to validate and demonstrate the documented logic.

The central editorial question is: If a content editor applies certain tags, where will that content appear?

Why this matters

  • Placement behavior is not always obvious to editors because it depends on Views, taxonomy, page context, and template-driven rules.
  • Clear documentation reduces guesswork and supports more consistent tagging.
  • A documented model also makes the site easier to maintain.

What has been accomplished

  • Documented the main placement mechanisms currently in use.
  • Separated descriptive taxonomy from placement-driving taxonomy.
  • Improved accuracy by accounting for related-content templates and inherited section context.
  • Created a supporting prototype and workbook to test and explain placements.

Placement model now documented

PatternDescription
Direct View-driven placementPlacements controlled by embedded Drupal Views with explicit filters or runtime arguments.
Template-driven related contentPlacements rendered by node templates or preprocess logic, such as related stories and related profiles.
Section inheritanceProgram and Center pages can imply a required context even when the rendered View does not expose explicit arguments.
Manual exceptionsA small number of sections require explicit override rules when the relationship is not fully exposed in configuration.

SourceMeaning

Manual overrideMaintained exception for URLs that require a fixed inherited term.
Supporting tools: The Content Placement Advisor and spreadsheet workbook should be treated as validation and demonstration tools for the documented rules, not as the primary deliverables.

Current state

  • Major placement rules have been identified and modeled.
  • Search views and modal-only views are excluded.
  • Related-content surfaces are separated from page-level placements.
  • Program and Center pages inherit section context more accurately.

Current limitations

  • Some site behavior still depends on conventions that are not fully explicit in Drupal configuration.
  • A small number of surfaces require manual override rules.
  • Results depend on the freshness of the crawl and exported datasets.
